# Documentation for Brush

This directory (`/docs/`) contains the technical documentation for the Brush project. It is designed to provide developers with the information they need to understand, contribute to, and use Brush effectively.

## Core Documentation

* [Getting Started](getting_started.md): Instructions on setting up the development environment, installing dependencies, and basic usage.
* [Core Concepts](core_concepts.md): Explanation of the fundamental concepts behind Brush, including Gaussian splatting and related principles.
* [Architecture](architecture.md): Detailed overview of the system architecture, components, and data flow.
* [Code Structure](code_structure.md): Description of the project's file organization, coding conventions, and key modules.
* [Key Technologies](key_technologies.md): In-depth information about the technologies used in Brush, such as Rust, WebGPU, and the Burn framework.

## Development and Contribution

* [Build Process](build_process.md): Instructions for building Brush from source for different platforms.
* [Development Workflow](development_workflow.md): Guidelines for contributing to the project, including Git workflow, code review, and branching strategies.
* [Debugging and Profiling](debugging_profiling.md): Techniques and tools for debugging and profiling Brush.
* [Testing](testing.md): Information about the testing strategy, different types of tests, and how to run them.

> **Note:** Information about AI-assisted development is available in the [project directory](/project/ai_assisted_development.md).
